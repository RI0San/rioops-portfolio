How it works — message flow

01
Message arrives via WhatsApp
Evolution API fires a webhook. Outgoing messages and delivery receipts are filtered out immediately.
webhook → Extract Message Data
02
Language detection + validation
Message is validated (no bot echoes, no empty payloads). Language is auto-detected — Arabic, French, or Darija — and the entire conversation follows that language.
Valid Message? → language detection
03
Business config loaded from Sheets
Per-client config (agency name, pricing rules, cars, tone) is fetched from Google Sheets — the client can update their own data without touching n8n.
Get Business Config (Supabase)
04
Customer memory + conversation history
Customer is retrieved or created in Supabase. Full conversation history is loaded so the AI always has context — no repeating questions.
Get or Create Customer → Get Conversation History
05
AI generates response + extracts intent
GPT-4o-mini reads the context, available cars, and conversation, then replies. A second code node extracts booking intent, negotiation signals, and date validity.
Prepare AI Context → AI Response Generator → Extract Intent
06
Confidence router branches the flow
A switch node routes on AI confidence: high → booking flow; medium → clarify; low → human escalation. No guesswork sent to the customer.
Validate Dates → Confidence Router
07
Booking created + owner notified if needed
If car is available, booking is written to Supabase and a confirmation is sent. If human escalation is triggered, owner gets a WhatsApp ping instantly.
Create Booking → Notify Owner → Save Conversations

How it works — message flow

01
Message arrives + validated
WhatsApp message hits Evolution API webhook. Noise (receipts, outgoing) is filtered immediately by the Extract Message Data node.
webhook → Extract Message Data → Valid Message?
02
Business config from Google Sheets
Vehicle catalog, pricing, and tone-of-voice rules are fetched from a Google Sheet the client controls — zero technical knowledge required from them.
Get Business Info (Google Sheets)
03
Customer profile + conversation loaded
Customer is upserted in Supabase. Full message history retrieved so the AI remembers everything — vehicle preferences, visit history, past objections.
Get or Create Customer → Get Conversation History → Get Available Vehicles
04
RAG semantic search runs
The last mentioned vehicle is detected, then a vector similarity search (pgvector) finds the most relevant past conversations and product knowledge to enrich the AI context.
Detect Last Vehicle → RAG system start → Process RAG Results
05
Luxury AI generates response
GPT-4o receives the full enriched context (business info + RAG results + history + vehicles) and generates a response calibrated for high-end Arabic sales.
Prepare Luxury AI Context → Luxury AI Response Generator
06
Intent extraction + customer intelligence
Two dedicated code nodes extract booking intent, requested PDF/brochure flags, customer preferences, and update the customer profile with every new signal.
Extract AI Response & Intents → Extract Customer Intelligence
07
Booking, PDF delivery + full data save
If booking intent → create in Supabase. If PDF requested → brochure sent via WhatsApp from Cloudflare R2. Conversation saved, customer record updated, response delivered.
Has Booking Intent? → Create Booking → Should Send PDF? → send pdf → Save Conversations → Send WhatsApp Response
